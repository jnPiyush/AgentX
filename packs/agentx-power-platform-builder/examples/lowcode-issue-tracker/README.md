# Example: AgentX Issue Tracker (Low-Code)

A minimal Power Platform solution generated by the low-code-builder agent. Imports cleanly into a Dataverse environment and works end-to-end in under 10 minutes.

## What This Solution Includes

| Component | Schema Name | Notes |
|-----------|-------------|-------|
| Table | `agx_issue` | Issue tracking table |
| Column (primary) | `agx_name` | Title |
| Column | `agx_description` | Multiline text |
| Column | `agx_status` | Choice: Open / In Progress / Closed |
| Column | `agx_priority` | Choice: Low / Medium / High |
| Column | `agx_assignedto` | Lookup -> systemuser |
| Column | `agx_duedate` | DateOnly |
| Flow | `agx_NotifyOnIssueCreated` | Sends an Outlook email when a new `agx_issue` row is created |
| Connection Reference | `agx_sharedoffice365` | Bound to Outlook on import |

## Prerequisites

- A Power Platform environment with Dataverse provisioned (trial works)
- The maker's Microsoft 365 account has access to that environment
- `pac` CLI installed:
  `
  dotnet tool install --global Microsoft.PowerApps.CLI.Tool
  pac --version
  `

## The 3-Command Import Flow

From this directory (`examples/lowcode-issue-tracker/`):

`
pac auth create --environment https://<your-env>.crm.dynamics.com --name agx-dev
pac solution pack --zipfile build/solution.zip --folder ./src --packagetype Unmanaged
pac solution import --path build/solution.zip --async --publish-changes --activate-plugins
`

On first import:

1. Open `make.powerapps.com`, switch to the target environment.
2. Open **Solutions** -> **AgentX Issue Tracker** -> **Connection references**.
3. Click `agx_sharedoffice365` and pick (or create) an Outlook connection.
4. Click **Publish all customizations**.

## Verify

1. In the maker portal, open the `agx_issue` table and click **+ New**.
2. Fill in a title, set priority/status, save.
3. Within a minute the cloud flow run history should show a successful send.
4. Check the configured inbox for the notification.

## Round-Trip Validation

To confirm the source tree round-trips through pac without importing:

`
pac solution pack --zipfile build/solution.zip --folder ./src --packagetype Unmanaged --allowDelete true
`

Exit code 0 means the tree is structurally valid.
