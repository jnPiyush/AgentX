Core Capabilities

Eight pillars that make AI delivery durable

AgentX is a Harness-Oriented Architecture. These eight capabilities turn AI generation into autonomous, auditable software delivery.

01

The Agentic Loop

Generate -> Verify -> Self-Review -> Fix -> repeat. The agent researches the repo, classifies the task, writes against clear criteria, and iterates until the work is definitively Done.

[loop complete] commit: feat: add /health (#142)

02

Model Council

High-stakes calls -- PRD scope, ADR options, AI design, code review, research -- get debated by an Analyst, Strategist, and Skeptic before they ship.

03

Deep Domain Skills

82 peer-reviewed skills loaded on demand. Repo-driven knowledge over model-memory guesses, with Karpathy guidelines wired into every coding agent.

04

Context Compaction

When prompt usage crosses 70% of the context window, AgentX preserves system rules, keeps recent turns verbatim, and summarizes the rest -- no context amnesia on long runs.

05

Self-Review & Validation Gates

Before every handoff, agents grade themselves on assumption audit, minimal-diff, and domain rules. HIGH and MEDIUM findings block the workflow until resolved.

06

Standardized Templates

PRDs, ADRs, Tech Specs, UX briefs, Reviews, Security Plans -- every artifact lands in a predictable structure so handoffs stay clean and the paper trail stays consistent.

07

Harness Engineering

The workspace is the state. Tracked execution plans, progress logs, memory files, and ADRs keep long-running agents durable, resumable, and grounded in repo reality.

08

Knowledge Compounding

Brainstorm and compound-loop entry points, ranked planning and review learnings, durable review-finding records, and one-step promotion of important findings into the backlog.

Discipline

Every change goes through the loop

AgentX scores every output against role-specific gates -- coverage, frontmatter, references, loop completion -- and refuses to hand off until they pass. No silent regressions, no skipped reviews.

Quality Loop

Iterative refinement until all gates pass. Self-review with HIGH/MEDIUM/LOW findings before any handoff.

Issue-First Workflow

Every change starts with an issue and routes through Status: Backlog -> Ready -> In Progress -> In Review -> Done.

Pre-Commit Gates

Secrets scan, blocked-command check, coverage threshold, loop completion -- enforced as git hooks, not vibes.

Evidence-Backed Validation

Every claim ("tests pass", "spec updated") cites a durable artifact in the repo -- plans, progress, contracts.

Extend

Bring your own agents and skills

AgentX is repo-driven. Drop a Markdown file into .github/agents/ or .github/skills/ and it shows up in the picker, the tree view, and the loop -- no rebuild, no plugin API.

Add an agent

A new role in five minutes

Define the role's contract -- description, model, handoffs, gates -- in one .agent.md file. Agent X picks it up automatically and routes matching work to it.

.github/agents/security-reviewer.agent.md 
---
description: Reviews PRs for OWASP Top 10 and supply-chain risks.
model: claude-opus-4.7
handoffs: [engineer, devops]
tools: [editFiles, runCommands, codebase]
---

# Security Reviewer Agent

You audit diffs against the OWASP Top 10 and the
repo's SECURITY-PLAN template. Block on HIGH findings.

## Done criteria
- All inputs validated at boundaries
- No hardcoded secrets, no string-concat SQL
- Dependencies scanned, results cited
  • Appears under @security-reviewer in chat the next time you reload.
  • Agent X auto-routes type:security issues to it.
  • Inherits the same loop, gates, and self-review contract as the built-ins.

Add a skill

Codify your team's playbook

Skills are progressive-disclosure Markdown files (under 5K tokens) that agents load on demand. Scaffold one with the skill-creator, then agents retrieve it whenever the description matches.

scaffold a new skill 
$ agentx skill new --category architecture --name event-driven
[ok] .github/skills/architecture/event-driven/SKILL.md
[ok] .github/skills/architecture/event-driven/scripts/
[ok] .github/skills/architecture/event-driven/references/

$ agentx skill validate event-driven
[PASS] frontmatter (name, description >= 50 chars)
[PASS] token budget (1.4K / 5K)
[PASS] structure (SKILL.md, optional scripts/refs)
[PASS] references resolve
Skill score: 38 / 40
  • Auto-indexed in Skills.md and the VS Code skill picker.
  • Agents load max 3-4 matching skills per task -- no context bloat.
  • Optional scripts/, references/, and assets/ ride along.
